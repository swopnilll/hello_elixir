# Elixir Learning Notes – Command Line Basics & First Project

# 1. Deck of Cards Simulator

We will build a program that simulates **handling playing cards**.

Examples of card games:
- Poker
- Blackjack
- Go Fish

But we **will NOT build a full game**.

Instead, we will implement common operations used when handling a deck of cards.

---

# 2. Important Card Terminology (For Beginners)

If you don’t play cards, here are simple explanations.

## Deck

A **deck** is a full set of playing cards.

Standard deck:

- 52 cards total

Cards have:

### Suits
- Hearts ♥
- Diamonds ♦
- Clubs ♣
- Spades ♠

### Values
- Ace
- 2 – 10
- Jack
- Queen
- King

Example card:

```
Ace of Spades
```

---

## Hand

A **hand** means:

> The cards given to a player from the deck.

Example:

If a player receives 5 cards from the deck:

```
Ace of Spades
7 of Hearts
King of Diamonds
2 of Clubs
Jack of Spades
```

This group of cards is called a **hand**.

So in programming terms:

```
Deck → full list of cards
Hand → subset of cards taken from the deck
```

# 3. Operations Our Program Will Implement

Our Elixir program will support these actions:

### 1. Create a Deck

Generate all 52 cards.

Example:

```
[
"Ace of Spades",
"2 of Spades",
"3 of Spades",
...
"King of Hearts"
]
```

---

### 2. Shuffle the Deck

Randomly reorder the cards.

Example:

Before shuffle:

```
Ace of Spades
2 of Spades
3 of Spades
...
```

After shuffle:

```
Queen of Hearts
4 of Clubs
Ace of Diamonds
...
```

---

### 3. Deal a Hand

Take some cards from the deck.

Example:

```
deal(deck, 5)
```

Result:

```
Hand → 5 cards
Remaining Deck → rest of the cards
```

---

### 4. Save Deck

Save the deck to the **file system**.

Example:

```
cards.txt
```

---

### 5. Load Deck

Read a deck back from a saved file.

# 6. Functional Programming vs Object-Oriented Programming

Most common languages use **Object-Oriented Programming (OOP)**.

Examples:

- Java
- Python
- Ruby
- JavaScript (mostly)

OOP focuses on:

```
Objects
Classes
Methods
State
```

Example idea:

```
Deck.shuffle()
Deck.deal()
```

---

## Elixir Uses Functional Programming

Elixir follows **functional programming**.

Instead of objects, we use:

- functions
- immutable data
- transformations

Example idea:

```
shuffle(deck)
deal(deck, 5)
```

The deck is **passed into functions and returned as new data**.
