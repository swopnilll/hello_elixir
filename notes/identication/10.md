# List.flatten, Enum.with_index & The Index Problem

## The Problem: Nested Lists

After `Enum.map(&mirror_row/1)`, we have a list of lists:

```elixir
[
  [145, 46, 200, 46, 145],
  [3, 178, 206, 178, 3],
  [73, 228, 165, 228, 73],
  ...
]
```

To iterate through this, we'd need **nested loops** - one to go through rows, another to go through elements.

## Solution: List.flatten

`List.flatten` converts a nested list into a single flat list:

```elixir
# Before (list of lists)
[[1, 2, 3], [4, 5, 6], [7, 8, 9]]

# After List.flatten
[1, 2, 3, 4, 5, 6, 7, 8, 9]
```

Now we can iterate through once with a single loop!

```elixir
def build_grid(%Identicon.Image{hex: hex} = image) do
  hex
  |> Enum.chunk(3)
  |> Enum.map(&mirror_row/1)
  |> List.flatten              # ← Flatten to single list
end
```

## The Index Problem in Elixir

### Elixir iteration has NO index by default!

Think back to our code:

```elixir
Enum.map(rows, &mirror_row/1)

def mirror_row(row) do   # ← Only gets the row, NOT the index!
  ...
end
```

The function has **no idea** if it's working on:
- The first row?
- The last row?
- A row in the middle?

### Why we need the index

When we draw the image, we need to know WHERE each square goes:

```
Index 0: top-left corner (0,0) to (50,50)
Index 1: next square (50,0) to (100,50)
Index 5: second row, first column (0,50) to (50,100)
...
```

Without the index, we can't position the squares!

## Solution: Enum.with_index

`Enum.with_index` adds the index to each element as a tuple:

```elixir
# Before
[145, 46, 200, 46, 145, 3, 178, ...]

# After Enum.with_index
[{145, 0}, {46, 1}, {200, 2}, {46, 3}, {145, 4}, {3, 5}, {178, 6}, ...]
#  ↑   ↑    ↑   ↑
# value index
```

Each element becomes `{value, index}`.

```elixir
def build_grid(%Identicon.Image{hex: hex} = image) do
  hex
  |> Enum.chunk(3)
  |> Enum.map(&mirror_row/1)
  |> List.flatten
  |> Enum.with_index           # ← Add index to each element
end
```

## Full Result

```elixir
iex> Identicon.main("asdf")
[
  {145, 0},   # First square, index 0
  {46, 1},    # Second square, index 1
  {200, 2},   # Third square, index 2
  ...
  {73, 24}    # Last square, index 24
]
```

25 elements total (0 to 24) - one for each square in our 5x5 grid!

## Updating the Struct

### Step 1: Add `grid` property to the struct

```elixir
# lib/image.ex
defmodule Identicon.Image do
  defstruct hex: nil, color: nil, grid: nil   # ← Added grid
end
```

### Step 2: Store the grid in build_grid

```elixir
def build_grid(%Identicon.Image{hex: hex} = image) do
  grid =
    hex
    |> Enum.chunk(3)
    |> Enum.map(&mirror_row/1)
    |> List.flatten
    |> Enum.with_index

  %Identicon.Image{image | grid: grid}
end
```

### Why `%Identicon.Image{image | grid: grid}`?

This syntax:
1. Takes all existing properties from `image` (hex, color)
2. Updates/adds the `grid` property
3. Returns a NEW struct (doesn't modify the original)

```elixir
# Equivalent to:
%Identicon.Image{
  hex: image.hex,
  color: image.color,
  grid: grid           # ← New value
}
```

## Code Style: Pipe Indentation

Convention in Elixir community:

```elixir
# Preferred style
grid =
  hex
  |> Enum.chunk(3)
  |> Enum.map(&mirror_row/1)
  |> List.flatten
  |> Enum.with_index

# Also acceptable
grid = hex
       |> Enum.chunk(3)
       |> Enum.map(&mirror_row/1)
```

Put the variable on its own line, indent the pipe chain.

## Final Output

```elixir
iex> Identicon.main("asdf")
%Identicon.Image{
  color: {145, 46, 200},
  grid: [{145, 0}, {46, 1}, {200, 2}, ... {73, 24}],
  hex: [145, 46, 200, 3, 178, 206, ...]
}
```

All our data in one struct:
- `hex` - original hash numbers
- `color` - RGB tuple for coloring
- `grid` - flattened list with indices for positioning

## Summary

| Function | What it does |
|----------|--------------|
| `List.flatten` | Converts nested list to single flat list |
| `Enum.with_index` | Adds index to each element as `{value, index}` |
| `%Struct{old \| key: new}` | Updates struct with new values |

## Why the Struct is Important

We're carrying around multiple pieces of data:
- `hex` - needed for color AND grid
- `color` - needed for drawing
- `grid` - needed for positioning squares

The struct keeps everything organized and accessible to any function in the pipeline!
