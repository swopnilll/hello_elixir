# Enum.map & Function References

## The Goal

Apply `mirror_row` to every row in our grid:

```
[[145, 46, 200], [3, 178, 206], ...]
              ↓ apply mirror_row to each
[[145, 46, 200, 46, 145], [3, 178, 206, 178, 3], ...]
```

## How Enum.map Works

`Enum.map` takes a list and a function. It:
1. Calls the function with each element
2. Collects all return values into a new list

```
┌─────────────────┐
│ Original List   │
│ [row1, row2...] │
└────────┬────────┘
         │
         ▼ for each element
    ┌─────────┐
    │ function│ ← mirror_row(row)
    └────┬────┘
         │
         ▼ collect results
┌─────────────────┐
│ New List        │
│ [result1, ...]  │
└─────────────────┘
```

## The Problem: Passing Function References

### Attempt 1: JavaScript style (FAILS)

```elixir
def build_grid(%Identicon.Image{hex: hex} = image) do
  hex
  |> Enum.chunk(3)
  |> Enum.map(mirror_row)   # ❌ ERROR!
end
```

**Error:** `undefined function mirror_row/0`

**Why?** In Elixir, writing `mirror_row` **calls** the function immediately. It doesn't pass a reference!

```elixir
# What we wrote:
Enum.map(mirror_row)

# What Elixir sees:
Enum.map(result_of_calling_mirror_row_with_no_args)
```

### Attempt 2: The & syntax (WORKS!)

```elixir
def build_grid(%Identicon.Image{hex: hex} = image) do
  hex
  |> Enum.chunk(3)
  |> Enum.map(&mirror_row/1)   # ✅ WORKS!
end
```

## The `&function/arity` Syntax

To pass a **reference** to a function (not call it), use:

```elixir
&function_name/number_of_arguments
```

| Part | Meaning |
|------|---------|
| `&` | "I'm passing a reference, not calling" |
| `mirror_row` | The function name |
| `/1` | The function takes 1 argument (arity) |

### Why `/1`?

Elixir allows multiple functions with the same name but different argument counts:

```elixir
def greet(), do: "Hello"           # greet/0
def greet(name), do: "Hello #{name}"   # greet/1
def greet(first, last), do: "Hello #{first} #{last}"  # greet/2
```

The `/1` tells Elixir exactly which version you want.

## Visual Comparison: JavaScript vs Elixir

### JavaScript
```javascript
// Passing a reference
list.map(mirrorRow)          // Just use the name

// Calling the function
mirrorRow()                  // Add parentheses
```

### Elixir
```elixir
# Passing a reference
Enum.map(list, &mirror_row/1)    # Use & and /arity

# Calling the function
mirror_row(row)                   # Just use the name (auto-calls!)
```

**It's basically backwards from JavaScript!**

## The Complete Code

```elixir
def build_grid(%Identicon.Image{hex: hex} = image) do
  hex
  |> Enum.chunk(3)
  |> Enum.map(&mirror_row/1)
end

def mirror_row(row) do
  [first, second | _tail] = row
  row ++ [second, first]
end
```

## Step-by-Step Execution

```elixir
hex = [145, 46, 200, 3, 178, 206, 73, 228, 165, 65, 6, 141, 73, 90, 181, 112]

# Step 1: Enum.chunk(hex, 3)
[[145, 46, 200], [3, 178, 206], [73, 228, 165], [65, 6, 141], [73, 90, 181]]

# Step 2: Enum.map(..., &mirror_row/1)
# Calls mirror_row([145, 46, 200]) → [145, 46, 200, 46, 145]
# Calls mirror_row([3, 178, 206])   → [3, 178, 206, 178, 3]
# ... for each row

# Final result:
[
  [145, 46, 200, 46, 145],
  [3, 178, 206, 178, 3],
  [73, 228, 165, 228, 73],
  [65, 6, 141, 6, 65],
  [73, 90, 181, 90, 73]
]
```

## Key Takeaways

1. **`Enum.map`** - Transforms every element in a list using a function
2. **Function names auto-call** - `mirror_row` calls it, doesn't reference it
3. **Use `&` for references** - `&mirror_row/1` passes a reference
4. **Arity matters** - `/1` specifies which version (1 argument)
5. **Immutability** - `map` creates a NEW list, doesn't modify the original

## Remember: No Mutation!

```elixir
original = [[1,2,3], [4,5,6]]
result = Enum.map(original, &mirror_row/1)

# original is UNCHANGED
# result is a NEW list
```

Elixir never modifies existing data - it creates new transformed versions.
