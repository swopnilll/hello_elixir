# Structs in Elixir

## The Problem

Our app has multiple pieces of data flowing through it:

- Input string
- List of hash numbers (hex)
- RGB color value
- Grid representation

Passing all this as separate arguments gets messy. We need a single data structure to hold it all.

## What is a Struct?

A **struct** is a special kind of map used to store data in Elixir applications.

### Struct vs Map

| Feature | Map | Struct |
|---------|-----|--------|
| Can have any keys | Yes | No - only defined properties |
| Default values | No | Yes |
| Compile-time checking | No | Yes - catches typos |

## Creating a Struct

### Step 1: Create a new module file

```
lib/image.ex
```

### Step 2: Define the struct

```elixir
defmodule Identicon.Image do
  defstruct hex: nil
end
```

- `defstruct` defines the struct
- `hex: nil` is a property with default value `nil`

### Step 3: Use the struct

```elixir
# Create with default values
%Identicon.Image{}
# => %Identicon.Image{hex: nil}

# Create with initial value
%Identicon.Image{hex: [145, 46, 200, 3, ...]}
# => %Identicon.Image{hex: [145, 46, 200, 3, ...]}
```

## Struct Enforcement

Structs only allow defined properties:

```elixir
# This works
%Identicon.Image{hex: [1, 2, 3]}

# This FAILS - mystery_prop not defined
%Identicon.Image{mystery_prop: "hello"}
# => ** (KeyError) key :mystery_prop not found
```

Maps don't have this protection - they accept any key.

## Using Struct in Our App

### Before (returning raw list)

```elixir
def hash_input(input) do
  :crypto.hash(:md5, input)
  |> :binary.bin_to_list
end
```

### After (returning struct)

```elixir
def hash_input(input) do
  hex = :crypto.hash(:md5, input)
        |> :binary.bin_to_list

  %Identicon.Image{hex: hex}
end
```

### Testing it

```elixir
iex> Identicon.main("asdf")
%Identicon.Image{hex: [145, 46, 200, 3, 178, 206, ...]}
```

## Important: Structs Are NOT Objects

Coming from OOP (Ruby, Python, JS)?

**Do NOT think of structs as classes with methods.**

| OOP Class | Elixir Struct |
|-----------|---------------|
| Has properties | Has properties |
| Has methods | **NO methods** |
| Creates instances | Just a map with rules |

```elixir
# You CANNOT do this:
image.get_color()      # NO - structs have no methods

# You DO this instead:
get_color(image)       # Pass struct to a function
```

**Structs are just maps with compile-time guarantees.** They hold primitive data only. Functions that operate on structs live in separate modules.

## Why "hex"?

The list of numbers represents hexadecimal values from the MD5 hash. Could also be called `seed` or `hash_values`.

## Summary

- Struct = Map with defined properties + default values + compile-time checks
- Created with `defstruct` inside a module
- Perfect for holding application data
- **NOT objects** - no methods, just data
