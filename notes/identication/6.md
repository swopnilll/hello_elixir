# Pattern Matching with Structs & Lists

## Goal: Pick Color Function

Extract the first 3 values from the hex list (RGB values).

```elixir
def main(input) do
  input
  |> hash_input
  |> pick_color
end
```

## The Journey: From Wrong to Right

### Attempt 1: JavaScript style - Dot notation + Bracket access

```elixir
def pick_color(image) do
  hex = image.hex        # ✅ This works! Gets the hex list
  r = hex[0]             # ❌ NOPE - no bracket notation in Elixir
  g = hex[1]             # ❌
  b = hex[2]             # ❌
end
```

**Elixir has no `list[index]` syntax.**

### Attempt 2: Map-style bracket access

```elixir
def pick_color(image) do
  hex = image["hex"]     # ❌ NOPE - structs don't support bracket access
end
```

**Structs are not maps.** You can't use `image["hex"]` syntax.

### Attempt 3: Dot notation works, but then what?

```elixir
def pick_color(image) do
  hex = image.hex        # ✅ Works - gets [145, 46, 200, 3, ...]
  hex                    # But how do we get first 3 elements?
end
```

Dot notation works for accessing struct properties, but we still can't index into the list.

### Attempt 4: Pattern matching the struct

```elixir
def pick_color(image) do
  %Identicon.Image{hex: hex_list} = image
  hex_list               # ✅ Works - same as image.hex
end
```

Pattern matching extracts `hex` into `hex_list`. But we still need the first 3 values.

### Attempt 5: Pattern matching the list (FAILS)

```elixir
def pick_color(image) do
  %Identicon.Image{hex: hex_list} = image
  [r, g, b] = hex_list   # ❌ ERROR: no match!
  [r, g, b]
end
```

**Error:** `no match of right hand side value: [145, 46, 200, 3, 178, ...]`

Why? Pattern `[r, g, b]` claims the list has exactly 3 elements. But it has 16!

### Attempt 6: The `| _rest` syntax (WORKS!)

```elixir
def pick_color(image) do
  %Identicon.Image{hex: hex_list} = image
  [r, g, b | _rest] = hex_list   # ✅ WORKS!
  [r, g, b]
end
```

The `|` pipe says: "Give me first 3, throw away the rest"

```
[r, g, b | _rest] = [145, 46, 200, 3, 178, 206, ...]

r     = 145
g     = 46
b     = 200
_rest = [3, 178, 206, ...]  (ignored)
```

### Attempt 7: Combine into one pattern (FINAL)

```elixir
def pick_color(image) do
  %Identicon.Image{hex: [r, g, b | _rest]} = image
  [r, g, b]
end
```

Nest the list pattern inside the struct pattern. One line does everything!

## Adding Type Annotation in Function Parameter

### Final version with type check

```elixir
def pick_color(%Identicon.Image{} = image) do
  %Identicon.Image{hex: [r, g, b | _rest]} = image
  [r, g, b]
end
```

### What does `%Identicon.Image{} = image` mean?

```elixir
def pick_color(%Identicon.Image{} = image) do
```

This is **pattern matching in the function parameter**:

| Part | Meaning |
|------|---------|
| `%Identicon.Image{}` | "The argument must be an Identicon.Image struct" |
| `= image` | "And bind it to the variable `image`" |

**Benefits:**

1. **Type safety** - Function only accepts `Identicon.Image` structs
2. **Clear intent** - Anyone reading knows what type is expected
3. **Early failure** - If wrong type passed, fails immediately with clear error

```elixir
# This works
pick_color(%Identicon.Image{hex: [1,2,3]})

# This fails immediately
pick_color(%{hex: [1,2,3]})           # ❌ Not an Identicon.Image
pick_color("hello")                    # ❌ Not an Identicon.Image
```

## Summary: The Evolution

```elixir
def pick_color(%Identicon.Image{} = image) do
  # hex = image["hex"]              # ❌ Wrong - structs don't support bracket access
  # hex = image.hex                 # ✅ Works but can't index into list

  # %Identicon.Image{hex: hex_list} = image
  # [r,g,b | _rest] = hex_list      # ✅ Two-step pattern matching
  # [r,g,b]

  %Identicon.Image{hex: [r,g,b | _rest]} = image   # ✅ Combined - cleanest
  [r,g,b]
end
```

## The Underscore Prefix

| Variable | Meaning |
|----------|---------|
| `rest` | I will use this variable |
| `_rest` | I acknowledge this exists but won't use it |
| `_` | Completely ignore (anonymous) |

Using `_rest` prevents compiler warnings about unused variables.

## Key Takeaways

1. **No `list[0]`** - Elixir doesn't have bracket index access for lists
2. **No `struct["key"]`** - Structs don't support map-style bracket access
3. **`struct.key` works** - Dot notation works for struct properties
4. **Pattern must match** - `[a, b, c]` only matches exactly 3-element lists
5. **Use `| _rest`** - To get first N elements, ignore the rest
6. **Nest patterns** - Combine struct + list matching in one expression
7. **Type in params** - `%Struct{} = var` ensures correct type is passed
