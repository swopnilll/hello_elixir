# Identicon Requirements & Application Flow

## Key Requirement: Deterministic Generation

Identicons are **not randomly generated**. They are deterministic based on a string input.

```
"Billy" → always produces the same identicon
"Jane"  → always produces a different, but consistent identicon
"Billy" → same identicon as before
```

### Why Deterministic?

- **No storage needed**: Don't need to save the actual image file
- **Generate on-the-fly**: Create the identicon whenever needed using just the username
- **Space efficient**: Store only the string, not the image
- **Consistent**: Users always see the same identicon for the same input

## Application Flow

```
┌─────────┐      ┌───────────────────┐      ┌─────────────┐
│  Input  │  →   │  Generate         │  →   │   Output    │
│ String  │      │  Identicon        │      │   Image     │
└─────────┘      └───────────────────┘      └─────────────┘
   "Billy"         (the hard part)           save to disk
```

1. **Input**: Take a string (e.g., username)
2. **Process**: Generate the identicon (main challenge)
3. **Output**: Save image to hard disk
