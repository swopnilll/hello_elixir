# Building the Identicon: MD5 Hashing Approach

## Elixir Design Pattern Recap

Transform data through a series of small functions:

```
initial data → func1 → func2 → func3 → ... → final result
```

No single "mega function" that does everything. Each function handles one small transformation.

## The Pipeline

```
┌────────┐    ┌────────────┐    ┌────────────┐    ┌─────────────┐    ┌───────┐
│ String │ →  │ Compute    │ →  │ Pick       │ →  │ Build       │ →  │ Save  │
│        │    │ MD5 Hash   │    │ Color      │    │ Grid/Image  │    │ Image │
└────────┘    └────────────┘    └────────────┘    └─────────────┘    └───────┘
```

## The Key: MD5 Hashing

### What is Hashing?

When you hash a string, you get back a **series of numbers unique to that string**.

```
"Billy" → [114, 179, 2, 191, 41, ...]   (always the same)
"Jane"  → [23, 87, 142, 56, 201, ...]   (different numbers)
"Billy" → [114, 179, 2, 191, 41, ...]   (reproducible!)
```

### Why MD5?

- **Deterministic**: Same input always produces same output
- **Unique**: Different inputs produce different outputs
- **Fixed length**: Always returns the same number of values

### How We'll Use It

1. **Pick color**: Use some of the hash numbers to determine RGB values
2. **Determine colored squares**: Use hash numbers to decide which grid squares get colored
3. **Generate image**: Convert the grid into actual pixels

## Summary

The MD5 hash is the foundation of the entire application. It converts our input string into a reproducible set of numbers that drive all visual decisions.
