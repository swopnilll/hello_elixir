# Implementing build_pixel_map

## The Goal

Transform each grid tuple into coordinate pairs for drawing rectangles.

```
Grid:       [{46, 1}, {200, 2}, {46, 3}, ...]
                 ↓ calculate coordinates
Pixel Map:  [{{50,0}, {100,50}}, {{100,0}, {150,50}}, ...]
             top-left  bottom-right
```

## Using Enum.map with Pattern Matching

We need to:
1. Iterate over each tuple in the grid
2. Extract the index (ignore the code - we don't need it anymore)
3. Calculate top-left and bottom-right coordinates
4. Return a tuple of coordinate pairs

```elixir
def build_pixel_map(%Identicon.Image{grid: grid} = image) do
  pixel_map = Enum.map grid, fn {_code, index} ->
    # _code: we acknowledge it exists, but don't need it
    # index: the position we'll use for calculations
  end
end
```

### Why `_code`?

The code (hex value) already served its purpose:
- Used to determine the color ✓
- Used to filter odd/even squares ✓

Now we only need the **index** to calculate pixel positions.

## The Implementation

```elixir
def build_pixel_map(%Identicon.Image{grid: grid} = image) do
  pixel_map = Enum.map grid, fn {_code, index} ->
    horizontal = rem(index, 5) * 50
    vertical = div(index, 5) * 50

    top_left = {horizontal, vertical}
    bottom_right = {horizontal + 50, vertical + 50}

    {top_left, bottom_right}
  end

  %Identicon.Image{image | pixel_map: pixel_map}
end
```

## Step-by-Step Breakdown

### Step 1: Pattern match the tuple

```elixir
fn {_code, index} ->
#   ↑       ↑
#  ignore  use this
```

### Step 2: Calculate horizontal position

```elixir
horizontal = rem(index, 5) * 50
```

| Index | `rem(index, 5)` | `* 50` | Meaning |
|-------|-----------------|--------|---------|
| 0 | 0 | 0 | Column 0, 0px from left |
| 1 | 1 | 50 | Column 1, 50px from left |
| 2 | 2 | 100 | Column 2, 100px from left |
| 6 | 1 | 50 | Column 1, 50px from left |
| 12 | 2 | 100 | Column 2, 100px from left |

### Step 3: Calculate vertical position

```elixir
vertical = div(index, 5) * 50
```

| Index | `div(index, 5)` | `* 50` | Meaning |
|-------|-----------------|--------|---------|
| 0 | 0 | 0 | Row 0, 0px from top |
| 1 | 0 | 0 | Row 0, 0px from top |
| 6 | 1 | 50 | Row 1, 50px from top |
| 12 | 2 | 100 | Row 2, 100px from top |

### Step 4: Create coordinate tuples

```elixir
top_left = {horizontal, vertical}
bottom_right = {horizontal + 50, vertical + 50}

{top_left, bottom_right}  # Return as a tuple
```

## Example Walkthrough

For grid element `{46, 1}`:

```elixir

horizontal = rem(1, 5) * 50 = 1 * 50 = 50
vertical   = div(1, 5) * 50 = 0 * 50 = 0
```

## The Pattern: Enum.map with Anonymous Function

```elixir
Enum.map collection, fn element ->
  # Transform element
  # Return new value
end
```

Each returned value becomes an element in the new list.

## Summary

| Step | What happens |
|------|--------------|
| 1 | `Enum.map` iterates over grid |
| 2 | Pattern match extracts `index` from `{_code, index}` |
| 3 | Calculate `horizontal` with `rem(index, 5) * 50` |
| 4 | Calculate `vertical` with `div(index, 5) * 50` |
| 5 | Create `top_left` and `bottom_right` tuples |
| 6 | Return `{top_left, bottom_right}` |
| 7 | Update struct with `pixel_map` |

Now we have everything needed to draw the image!
