```elixir
def pick_color(%Identicon.Image{hex: [r, g, b | _]} = image) do
  #                                                 ^^^^^^^^

```

### draw_image (don't need the full struct)

```elixir
def draw_image(%Identicon.Image{color: color, pixel_map: pixel_map}) do

  # No `= image` - we're at the end, don't need the struct anymore

```

**Key insight:** `= image` is optional. Only use it when you need access to the whole struct.

## The EGD Library Functions

### Step 1: Create a blank canvas

```elixir
image = :egd.create(250, 250)
```

Creates a 250x250 pixel image in memory.

### Step 2: Create a color/fill object

```elixir
fill = :egd.color(color)
```

EGD needs a "color object" for drawing, not raw RGB values.

```elixir
# color = {145, 46, 200}
# fill = color object that EGD can use
```

### Step 3: Draw rectangles

```elixir
:egd.filledRectangle(image, start, stop, fill)
```

| Argument | What it is |
|----------|------------|
| `image` | The canvas we're drawing on |
| `start` | Top-left point `{x, y}` |
| `stop` | Bottom-right point `{x, y}` |
| `fill` | The color object |

### Step 4: Render to PNG format

```elixir
:egd.render(image)
```

Converts the in-memory image to actual PNG binary data.

## Enum.each vs Enum.map

### Enum.map

- Transforms each element
- **Returns a new list**
- Use when you need the results

### Enum.each

- Runs code for each element
- **Returns `:ok`** (ignores return values)
- Use for side effects (like drawing)

```elixir
# We don't need the return values, just draw each rectangle
Enum.each pixel_map, fn {start, stop} ->
  :egd.filledRectangle(image, start, stop, fill)
end
```

## Important: EGD Mutates In Place!

This is **unusual** in Elixir/Erlang:

```elixir
image = :egd.create(250, 250)

# This MODIFIES image directly (doesn't return new image)
:egd.filledRectangle(image, start, stop, fill)

# Returns :ok, not a new image!
```

Normally in Elixir, we'd do:
```elixir
image = :egd.filledRectangle(image, ...)  # Would return new image
```

But EGD is a rare exception - it mutates the original.

## The Complete draw_image Function

```elixir
def draw_image(%Identicon.Image{color: color, pixel_map: pixel_map}) do
  image = :egd.create(250, 250)
  fill = :egd.color(color)

  Enum.each pixel_map, fn {start, stop} ->
    :egd.filledRectangle(image, start, stop, fill)
  end

  :egd.render(image)
end
```

## Saving to Disk

### The save_image function

```elixir
def save_image(image, input) do
  File.write("#{input}.png", image)
end
```

### How it's called in the pipeline

```elixir
|> draw_image
|> save_image(input)
```

**Wait, how does this work?**

```elixir
# What it looks like:
save_image(input)

# What actually happens (pipe inserts first arg):
save_image(result_from_draw_image, input)
```

The pipe operator:
1. Takes return value from `draw_image` (the PNG binary)
2. Inserts it as **first argument** to `save_image`
3. `input` becomes the **second argument**

### String Interpolation

```elixir
"#{input}.png"
```

| If `input` is | Result is |
|---------------|-----------|
| `"asdf"` | `"asdf.png"` |
| `"banana"` | `"banana.png"` |
| `"myname"` | `"myname.png"` |

Syntax: `#{}` inside a string injects a variable's value.

## The Complete Flow

```
"asdf"
   ↓ hash_input
%Image{hex: [145, 46, 200, ...]}
   ↓ pick_color
%Image{hex: ..., color: {145, 46, 200}}
   ↓ build_grid
%Image{..., grid: [{145,0}, {46,1}, ...]}
   ↓ filter_odd_squares
%Image{..., grid: [{46,1}, {200,2}, ...]}  # Only even values
   ↓ build_pixel_map
%Image{..., pixel_map: [{{50,0}, {100,50}}, ...]}
   ↓ draw_image
<<PNG binary data>>
   ↓ save_image("asdf")
:ok  (file saved as asdf.png)
```

## Summary

| Function | Purpose |
|----------|---------|
| `:egd.create(w, h)` | Create blank canvas |
| `:egd.color(rgb)` | Create color object |
| `:egd.filledRectangle(img, p1, p2, fill)` | Draw rectangle |
| `:egd.render(img)` | Convert to PNG binary |
| `File.write(name, data)` | Save to disk |
| `Enum.each` | Iterate without collecting results |
| `"#{var}.png"` | String interpolation |

## Key Takeaways

1. **Pattern matching without binding** - Don't need `= image` if you won't use the whole struct
2. **Enum.each** - For side effects, when you don't need return values
3. **EGD mutates** - Rare exception to Elixir's immutability
4. **Pipe with extra args** - `|> func(arg)` becomes `func(pipe_result, arg)`
5. **String interpolation** - `"#{var}"` injects variable into string
