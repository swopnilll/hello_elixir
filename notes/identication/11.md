# Filtering Odd Squares: Enum.filter & Anonymous Functions

## The Goal

Remove all odd-numbered squares from the grid (only keep even = colored squares).

```
Before: [{145, 0}, {46, 1}, {200, 2}, {46, 3}, {145, 4}, ...]
                    ↑        ↑        ↑
                   even     even     even

After:  [{46, 1}, {200, 2}, {46, 3}, ...]  # Only even values kept
```

## Enum.filter

`Enum.filter` keeps elements where the function returns `true`:

```elixir
Enum.filter(list, fn element ->
  # return true to keep, false to remove
end)
```

## Anonymous Functions (fn syntax)

### Two ways to pass functions

**Method 1: Named function with reference (what we did before)**

```elixir
def mirror_row(row), do: ...

Enum.map(rows, &mirror_row/1)
```

**Method 2: Anonymous function (inline)**

```elixir
Enum.filter(grid, fn square ->
  # logic here
end)
```

### Anonymous function syntax

```elixir
fn argument -> body end
```

| Part | Purpose |
|------|---------|
| `fn` | Start of anonymous function |
| `argument` | Parameter(s) the function receives |
| `->` | Separates arguments from body |
| `body` | The function logic |
| `end` | End of anonymous function |

### Comparison to other languages

```javascript
// JavaScript
(square) => { return square % 2 === 0; }
```

```ruby
# Ruby
{ |square| square % 2 == 0 }
```

```elixir
# Elixir
fn square -> rem(square, 2) == 0 end
```

## Pattern Matching in Anonymous Functions

Our grid contains tuples: `{value, index}`

We only care about the value, not the index:

```elixir
# Instead of receiving the whole tuple
fn square -> ... end

# Pattern match to extract just what we need
fn {code, _index} -> ... end
#   ↑      ↑
#  value  ignored
```

## The rem Function (Remainder)

`rem(number, divisor)` returns the remainder after division:

```elixir
iex> rem(20, 2)
0           # 20 ÷ 2 = 10 remainder 0 (even)

iex> rem(21, 2)
1           # 21 ÷ 2 = 10 remainder 1 (odd)

iex> rem(145, 2)
1           # odd

iex> rem(46, 2)
0           # even
```

**Rule:**
- `rem(n, 2) == 0` → even number
- `rem(n, 2) == 1` → odd number

## Important: Zero is NOT Falsy in Elixir!

Unlike JavaScript/Ruby, Elixir doesn't treat `0` as false:

```elixir
iex> 0 == false
false              # 0 is NOT falsy in Elixir!
```

So we MUST explicitly compare:

```elixir
# Wrong - won't work as expected
fn {code, _} -> rem(code, 2) end

# Correct - explicit comparison
fn {code, _} -> rem(code, 2) == 0 end
```

## The Complete Code

```elixir
def filter_odd_squares(%Identicon.Image{grid: grid} = image) do
  grid = Enum.filter grid, fn {code, _index} ->
    rem(code, 2) == 0
  end

  %Identicon.Image{image | grid: grid}
end
```

### Code style: Removing parentheses

When the last argument is a function, we can drop parentheses for cleaner code:

```elixir
# With parentheses (cluttered)
Enum.filter(grid, fn {code, _index} -> rem(code, 2) == 0 end)

# Without parentheses (cleaner)
Enum.filter grid, fn {code, _index} ->
  rem(code, 2) == 0
end
```

## Step-by-Step Execution

```elixir
grid = [{145, 0}, {46, 1}, {200, 2}, {46, 3}, {145, 4}, ...]

# Enum.filter checks each element:
{145, 0} → rem(145, 2) == 0 → 1 == 0 → false → REMOVED
{46, 1}  → rem(46, 2) == 0  → 0 == 0 → true  → KEPT
{200, 2} → rem(200, 2) == 0 → 0 == 0 → true  → KEPT
{46, 3}  → rem(46, 2) == 0  → 0 == 0 → true  → KEPT
{145, 4} → rem(145, 2) == 0 → 1 == 0 → false → REMOVED
...

# Result: only even-valued tuples remain
```

## Pipeline Pattern: Always Return Updated Struct

Every helper function should return an updated struct:

```elixir
def filter_odd_squares(%Identicon.Image{grid: grid} = image) do
  grid = Enum.filter grid, fn {code, _index} ->
    rem(code, 2) == 0
  end

  %Identicon.Image{image | grid: grid}  # ← Always return struct!
end
```

This keeps the pipeline flowing:

```elixir
input
|> hash_input           # returns struct
|> pick_color           # returns struct
|> build_grid           # returns struct
|> filter_odd_squares   # returns struct
|> draw_image           # next step...
```

## Summary

| Concept | Syntax |
|---------|--------|
| Anonymous function | `fn arg -> body end` |
| Pattern match in fn | `fn {value, _ignore} -> ... end` |
| Remainder | `rem(number, 2)` |
| Check even | `rem(n, 2) == 0` |
| Filter list | `Enum.filter(list, fn ... end)` |

## Key Takeaways

1. **`Enum.filter`** - Keeps elements where function returns `true`
2. **Anonymous functions** - `fn arg -> body end` for inline functions
3. **Pattern matching in fn** - Extract only what you need from complex data
4. **`rem/2`** - Calculate remainder (useful for even/odd check)
5. **Zero is NOT falsy** - Must use explicit `== 0` comparison
6. **Always return struct** - Keep the pipeline flowing!
