# Implementing the Hash Function & Grid Logic

## Code Structure

```elixir
defmodule Identicon do
  def main(input) do
    input
    |> hash_input
  end

  def hash_input(input) do
    :crypto.hash(:md5, input)
    |> :binary.bin_to_list
  end
end
```

## Understanding the Hash Code

```elixir
:crypto.hash(:md5, input)
|> :binary.bin_to_list
```

### What Each Part Does

| Code | What it does |
|------|--------------|
| `:crypto.hash(:md5, input)` | Converts string into a unique binary hash (raw bytes) |
| `:binary.bin_to_list` | Converts those bytes into a list of numbers (0-255) |

### Simple Explanation

Think of it like this:

```
"banana" → [114, 179, 2, 191, 41, 122, 34, 138, 117, 115, ...]
```

1. **`:crypto.hash(:md5, input)`** - Takes your string and runs it through the MD5 algorithm. This produces a "fingerprint" of your string as raw binary data. The same string ALWAYS produces the same fingerprint.

2. **`:binary.bin_to_list`** - The binary data isn't easy to work with, so we convert it to a list of 16 numbers (each between 0-255). Now we have numbers we can use!

### Why MD5? (Based on Requirements)

| Requirement | How MD5 Solves It |
|-------------|-------------------|
| Same input = same identicon | MD5 is **deterministic** - "Billy" always hashes to the same numbers |
| Different inputs = different identicons | MD5 produces **unique** outputs for different strings |
| Need numbers for color/grid | MD5 gives us **16 numbers** between 0-255 |
| RGB colors need 0-255 values | MD5 numbers are already in the perfect range! |

## Using the Numbers

Given input `"asdf"` produces: `[145, 46, 200, 3, 178, 206, ...]`

### 1. Pick Color (First 3 Numbers)

```
R = 145
G = 46
B = 200
```

### 2. Build the Grid

The 5x5 grid is mirrored, so we only need 3 columns of data:

```
Index:  1   2   3   2   1
        4   5   6   5   4
        7   8   9   8   7
       10  11  12  11  10
       13  14  15  14  13
```

Map numbers to positions (mirror columns 1-2 to columns 4-5):

```
145  46  200  46  145
  3 178  206 178    3
...
```

### Clarification: Index vs Numbers

**Important:** Index and Numbers serve different purposes!

| Concept | Purpose |
|---------|---------|
| **Index (1, 2, 3...)** | WHERE the square is located (position) |
| **Number (145, 46, 200...)** | WHETHER to color that square (even/odd test) |

**Visual example:**

```
Grid Position:     What number is there:     Colored? (even=yes):

  [1] [2] [3]        145  46  200             [ ] [X] [X]
       ↓                  ↓                        ↓
  Position 2         Number 46              46 is even → colored!
```

**How the 16 hash numbers are used:**

```
Hash: [145, 46, 200, 3, 178, 206, 73, 228, 165, 65, 6, 141, 73, 90, 181, 112]
       ↑    ↑    ↑   ↑   ↑    ↑   ...
       │    │    │   └───┴────┴── Used for grid positions 4, 5, 6, etc.
       └────┴────┴────────────── Used for BOTH:
                                  • RGB color (R=145, G=46, B=200)
                                  • Grid positions 1, 2, 3
```

The first 3 numbers do double duty: they define the ONE color for the entire identicon AND they fill grid positions 1, 2, 3. All remaining numbers (4th onwards) fill the rest of the grid positions.

### 3. Color Rule: Even = Colored, Odd = White

```
145 (odd)  → white
 46 (even) → colored
200 (even) → colored
 46 (even) → colored
145 (odd)  → white
```

Result for first row: `[ ] [X] [X] [X] [ ]`
