# Pattern Matching in Elixir (Notes)

## 🧠 Core Idea

* **Pattern matching = Elixir’s way of assigning variables**
* The `=` operator is **not just assignment** — it tries to **match structure on both sides**

---

## 🔹 Tuple Example (Cards Deal)

```elixir
{hand, rest_of_deck} = Cards.deal(deck, 5)
```

### What’s happening:

* `Cards.deal/2` returns a **tuple**

  ```elixir
  {hand, remaining_cards}
  ```
* Left side `{hand, rest_of_deck}` mirrors the structure
* Elixir:

  * Matches first element → assigns to `hand`
  * Matches second element → assigns to `rest_of_deck`

✅ Result:

```elixir
hand          # first 5 cards
rest_of_deck  # remaining cards
```

---

## 🔹 Key Rule

> Left side must **match the structure and size** of the right side

* Same type (tuple/list)
* Same number of elements

Otherwise → ❌ **Error**

---

## 🔹 List Pattern Matching

### Full list assignment

```elixir
colors = ["red"]
```

* `colors` gets entire list

---

### Extract single value

```elixir
[color] = ["red"]
```

* Matches list with 1 element
* `color = "red"`

---

### Extract multiple values

```elixir
[color1, color2] = ["red", "blue"]
```

Result:

```elixir
color1 = "red"
color2 = "blue"
```

---

### Mismatch example (Error ❌)

```elixir
[color1, color2, color3] = ["red", "blue"]
```

* Left expects 3 elements
* Right has only 2

➡️ Pattern does not match → **crash**

---

## 🔹 Mental Model

Think of pattern matching as:

1. Look at **right side value**
2. Copy its **structure on the left**
3. Assign values **position by position**

---

## 🔹 Important Takeaways

* `=` means **match**, not just assign
* Works with:

  * Tuples
  * Lists
  * Many other structures (later)
* Structure must match exactly
* Very **core concept in Elixir**

---

## 🔹 Why It Matters

* Clean way to extract values
* No need for index-based access
* Safer (forces structure correctness)
* Used everywhere in Elixir
