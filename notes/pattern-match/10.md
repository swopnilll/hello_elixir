# Updating Maps in Elixir (Notes)

---

## 🧠 Core Rule (VERY IMPORTANT)

> ❗ You **cannot mutate data** in Elixir
> ✅ You always create a **new copy**

---

## 🔹 Example Map

```elixir id="u1"
colors = %{primary: "red"}
```

---

## ❌ What You Might Try (Wrong)

```elixir id="u2"
colors.primary = "blue"
```

🚫 Error!

👉 Why?

* Elixir does NOT allow mutation

---

## 🔹 Correct Approach

> Create a **new map** with updated value

---

# ✅ Method 1: Using `Map.put/3`

```elixir id="u3"
new_colors = Map.put(colors, :primary, "blue")
```

---

## 🔍 Result

```elixir id="u4"
new_colors   # %{primary: "blue"}
colors       # %{primary: "red"}
```

👉 Original map unchanged ✅

---

## 🔹 Add New Key

```elixir id="u5"
Map.put(colors, :secondary, "green")
```

---

## 🔹 Key Points

* Works for:

  * Updating existing key
  * Adding new key
* Always returns **new map**

---

# ✅ Method 2: Update Syntax (Shorthand)

```elixir id="u6"
new_colors = %{colors | primary: "blue"}
```

---

## 🔍 Result

```elixir id="u7"
%{primary: "blue"}
```

---

## ⚠️ BIG GOTCHA

```elixir id="u8"
%{colors | secondary: "green"}
```

❌ ERROR!

👉 Why?

* Key `:secondary` does NOT exist

---

## 🧠 Rule

> This syntax ONLY updates existing keys

---

## 🔹 Summary of Both Methods

| Method      | Can Update | Can Add | Notes          |                      |
| ----------- | ---------- | ------- | -------------- | -------------------- |
| `Map.put/3` | ✅          | ✅       | Safe, flexible |                      |
| `%{map      | key: v}`   | ✅       | ❌              | Cleaner, but limited |

---

## 🔹 When to Use What?

* Use `Map.put` → when unsure if key exists
* Use `%{}` syntax → when key definitely exists

---

## 🧠 Mental Model

```text
Old Map → Copy → Modify → New Map
```

---

## 🔥 Important Insight

> Elixir favors **immutability**

* No side effects
* Safer code
* Easier debugging

---

## 🔹 Bonus: Ignore Original

```elixir id="u9"
colors = Map.put(colors, :primary, "blue")
```

👉 Now you overwrite variable (common pattern)

---

## 🧠 Takeaways

* Maps cannot be changed directly
* Always return new version
* Two update styles:

  * Function-based
  * Syntax-based
* Understand the limitation of syntax update
