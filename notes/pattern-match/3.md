# Saving Data in Elixir (Using Erlang) — Notes

## 🧠 Goal

* Save a **deck (list of strings)** to a file
* Use it later by loading from disk

---

## 🔹 Function Signature

```elixir
def save(deck, file_name) do
end
```

### Inputs:

* `deck` → list of cards
* `file_name` → name of file to save

---

## 🔹 Implementation

```elixir
def save(deck, file_name) do
  binary = :erlang.term_to_binary(deck)
  File.write(file_name, binary)
end
```

---

## 🔍 Step-by-Step

### 1️⃣ Convert Data → Binary

```elixir
binary = :erlang.term_to_binary(deck)
```

* Calls **Erlang function**
* Converts Elixir data → **storable format**
* Not raw 0s/1s, but a safe internal binary format

---

### 2️⃣ Write to File

```elixir
File.write(file_name, binary)
```

* Writes binary data to file
* Uses Elixir’s standard library

---

## 🔹 Calling Erlang Code

```elixir
:erlang.term_to_binary(deck)
```

### Key idea:

* `:` → access Erlang module
* `erlang` → Erlang module name

✅ Format:

```elixir
:module.function(args)
```

---

## 🔹 Output

```elixir
:ok
```

* Indicates success
* Common pattern in Elixir/Erlang

---

## 📁 Result

* File created in project directory:

  ```
  my_deck
  ```

* Contents:

  * Looks partially readable
  * Contains encoded data (not plain text)

---

## ⚠️ Important Notes

* File has:

  * Some readable strings
  * Some encoded/binary content
* Not meant for manual editing

---

## 🧠 Why Use `term_to_binary`?

* Ensures:

  * Safe storage
  * Accurate reconstruction later
* Works with:

  * Lists
  * Tuples
  * Maps
  * Any Elixir term

---

## 🔹 Mental Model

```
Elixir Data → Erlang Binary → File System
```

## 🧠 Takeaways

* Elixir can call Erlang directly
* `:erlang.term_to_binary` prepares data for storage
* `File.write` saves it
* Result is a reusable saved state
