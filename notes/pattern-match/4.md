# Loading Data + Pattern Matching + Error Handling (Elixir Notes)

---

## 🧠 Goal

* Load saved deck from file
* Decode binary → original list
* Handle errors cleanly (Elixir way)

---

## 🔹 Step 1: Read File

```elixir id="x1a2b3"
File.read("my_deck")
```

### Returns (IMPORTANT):

```elixir id="x4c5d6"
{:ok, binary_data}
```

OR

```elixir id="x7e8f9"
{:error, reason}
```

---

## 🧠 Key Pattern

> Elixir often returns tuples like:

```elixir id="x10"
{:ok, result}
{:error, reason}
```

* First element → status
* Second element → actual data

---

## 🔹 Step 2: Extract Values (Pattern Matching)

```elixir id="x11"
{status, binary} = File.read(file_name)
```

* `status` → `:ok` or `:error`
* `binary` → actual file data

---

## 🔹 Step 3: Convert Binary → Original Data

```elixir id="x12"
:erlang.binary_to_term(binary)
```

* Reverse of `term_to_binary`
* Returns original list of cards

---

## ✅ Basic Load Function (Initial Version)

```elixir id="x13"
def load(file_name) do
  {status, binary} = File.read(file_name)
  :erlang.binary_to_term(binary)
end
```

---

## ⚠️ Problem

If file doesn’t exist:

```elixir id="x14"
{:error, :enoent}
```

Then:

```elixir id="x15"
:erlang.binary_to_term(binary)
```

❌ CRASHES (because binary is invalid)

---

## ❌ JavaScript Approach (What NOT to do in Elixir)

```javascript id="x16"
const result = readFile(fileName);

if (result.status === "ok") {
  return decode(result.data);
} else {
  return "File not found";
}
```

### Problems:

* Manual checking
* Imperative style
* Not idiomatic in Elixir

---

## ✅ Elixir Approach (Pattern Matching + Case)

```elixir id="x17"
def load(file_name) do
  {status, binary} = File.read(file_name)

  case status do
    :ok ->
      :erlang.binary_to_term(binary)

    :error ->
      "That file does not exist"
  end
end
```

---

## 🔍 How `case` Works

```elixir id="x18"
case value do
  pattern1 -> result1
  pattern2 -> result2
end
```

### In our case:

* Check `status`
* Match:

  * `:ok`
  * `:error`

---

## 🔹 Real Flow

```elixir id="x19"
File.read(file_name)
→ {:ok, binary}
→ matches :ok
→ decode binary
```

OR

```elixir id="x20"
File.read(file_name)
→ {:error, reason}
→ matches :error
→ return message
```

---

## 🧠 Why This is Powerful

* No manual `if` checks
* Structure-driven logic
* Safer (forces handling cases)

---

## 🔹 Mental Shift (IMPORTANT)

| JavaScript Thinking       | Elixir Thinking              |
| ------------------------- | ---------------------------- |
| Check conditions manually | Match patterns automatically |
| `if / else`               | `case + pattern matching`    |
| Mutable flow              | Declarative flow             |

---

## 🔹 Even Better Pattern (Preview)

Instead of this:

```elixir id="x21"
{status, binary} = File.read(file_name)
```

We can match directly inside `case` (cleaner 👇)

```elixir id="x22"
case File.read(file_name) do
  {:ok, binary} ->
    :erlang.binary_to_term(binary)

  {:error, _reason} ->
    "That file does not exist"
end
```

---

## 🔥 Why This is Better

* No intermediate variables
* Pattern matching happens **inside case**
* Cleaner + more idiomatic

---

## 🧠 Takeaways

* File reading returns tuples (`:ok`, `:error`)
* Pattern matching extracts values
* Avoid `if` → prefer `case`
* Match structure, not conditions
* Elixir code = **data-driven logic**

---

## 🚀 Final Clean Version

```elixir id="x23"
def load(file_name) do
  case File.read(file_name) do
    {:ok, binary} ->
      :erlang.binary_to_term(binary)

    {:error, _reason} ->
      "That file does not exist"
  end
end
```
