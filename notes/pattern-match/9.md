# Maps in Elixir (Notes)

---

## 🧠 What is a Map?

> A **map** is a collection of **key–value pairs**

---

## 🔹 Similar To

| Language   | Equivalent  |
| ---------- | ----------- |
| JavaScript | Object `{}` |
| Ruby       | Hash `{}`   |

---

## 🔹 Creating a Map

```elixir id="m1"
colors = %{primary: "red"}
```

---

## 🔹 Multiple Key-Value Pairs

```elixir id="m2"
colors = %{primary: "red", secondary: "blue"}
```

---

## 🔹 Accessing Values

### 1️⃣ Dot Notation

```elixir id="m3"
colors.primary     # "red"
colors.secondary   # "blue"
```

---

### 2️⃣ Using Pattern Matching

```elixir id="m4"
%{secondary: secondary_color} = colors
```

```elixir id="m5"
secondary_color   # "blue"
```

---

## 🔍 How Pattern Matching Works (Maps)

```elixir id="m6"
%{secondary: secondary_color} = colors
```

### Step-by-step:

1. Check both sides are maps
2. Check key `:secondary` exists
3. Extract value → assign to `secondary_color`

---

## ⚠️ Important Rule

> Keys on the left MUST exist in the map

```elixir id="m7"
%{unknown: value} = colors
```

❌ Error (key doesn’t exist)

---

## 🔹 Why Pattern Matching is Powerful

* Extract only what you need
* Ignore rest of data
* Cleaner than manual access

---

## 🔹 Alternative (Less “Elixir-style”)

```elixir id="m8"
secondary_color = colors.secondary
```

---

## 🧠 Mental Model

```text id="m9"
Map → Match structure → Extract value
```

---

## 🔥 Key Insight

> Reading maps is easy
> Updating maps is where things get tricky ⚠️

---

## 🔹 Summary

* Maps = key-value data structure
* Created with `%{}`
* Access via:

  * dot notation
  * pattern matching
* Keys must exist when matching
