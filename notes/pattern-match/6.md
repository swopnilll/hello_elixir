# Creating a Helper Function + Introduction to Pipe Operator (Elixir Notes)

---

## 🧠 Problem

Users are repeatedly doing:

```elixir
deck = Cards.create_deck()
deck = Cards.shuffle(deck)
{hand, _rest} = Cards.deal(deck, hand_size)
```

👉 Same 3 steps every time:

1. Create deck
2. Shuffle
3. Deal

---

## 🎯 Goal

Create a **single function** to do all of this:

```elixir
Cards.create_hand(hand_size)
```

---

## 🔹 Function Signature

```elixir
def create_hand(hand_size) do
end
```

---

## 🔹 Step-by-Step Implementation (Without Pipe)

```elixir
def create_hand(hand_size) do
  deck = create_deck()
  deck = shuffle(deck)
  {hand, _rest} = deal(deck, hand_size)
  hand
end
```

---

## 🔍 What’s Happening

### 1️⃣ Create deck

```elixir
deck = create_deck()
```

---

### 2️⃣ Shuffle deck

```elixir
deck = shuffle(deck)
```

* Reassigns new shuffled deck

---

### 3️⃣ Deal cards

```elixir
{hand, _rest} = deal(deck, hand_size)
```

* Pattern matching:

  * `hand` → cards we want
  * `_rest` → ignored

---

### 4️⃣ Return result

```elixir
hand
```

---

## ⚠️ Problem with This Approach

* Repetitive (`deck` everywhere)
* Harder to read
* Not idiomatic Elixir

---

## 🧠 Pattern We Notice

```text
output → input → output → input → output
```

* Each function passes result to next
* Very common in Elixir

---

## 🔥 This Leads To: Pipe Operator

> Elixir has a special operator for this pattern

```elixir
|>
```

---

## 🧩 Conceptual Flow

Instead of:

```elixir
shuffle(deck)
deal(deck, hand_size)
```

We want:

```elixir
deck
→ shuffle
→ deal
```

---

## 🧠 Mental Model

> “Take result from previous step and pass it to next function”

---

## 🔹 Why This Matters

* Cleaner code
* Less repetition
* Easier to read
* Very common in real-world Elixir

---

## 🔜 Next Step

We will refactor this:

```elixir
deck = create_deck()
deck = shuffle(deck)
{hand, _rest} = deal(deck, hand_size)
```

👉 Into a **pipe-based version**

---

## 🧠 Takeaways

* Build helper functions for repeated logic
* Pattern matching extracts needed values
* Repetition signals need for better abstraction
* Pipe operator solves chaining problem

---

## 🚀 Big Idea

> Elixir prefers **data flowing through functions**
