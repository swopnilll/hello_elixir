# 📘 Elixir Learning Notes: Keyword Lists

## 🔹 What are Keyword Lists?

Keyword lists are a special data structure in Elixir that combine:

- Lists (ordered collection)
- Tuples (key-value pairs)
- Map-like access

A keyword list is essentially a **list of tuples**:

```elixir
[{:key, value}, {:key2, value2}]
```

Shortcut syntax:

```elixir
[key: value, key2: value2]
```

---

## 🔹 Example

```elixir
colors = [{:primary, "red"}, {:secondary, "green"}]
```

Shortcut version:

```elixir
colors = [primary: "red", secondary: "green"]
```

---

## 🔹 Accessing Values

Access values using square brackets:

```elixir
colors[:primary]
# => "red"
```

👉 Elixir searches through the list and returns the value for the first matching key.

---

## 🔹 Important: Internals

Even though keyword lists look like maps:

- They are **NOT maps**
- They are still **lists of tuples**

Example:

```elixir
[primary: "red"]
# is actually
[{:primary, "red"}]
```

---

## 🔹 Key Difference from Maps

### Maps:
- Keys must be **unique**
- Duplicate keys get overwritten

```elixir
%{primary: "red", primary: "blue"}
# => %{primary: "blue"}
```

### Keyword Lists:
- Keys **can be duplicated**

```elixir
[primary: "red", primary: "blue"]
# => [primary: "red", primary: "blue"]
```

---

## 🔹 Why Use Keyword Lists?

Best used when:

- You want **ordered data**
- You want **duplicate keys**
- You want **simple options/config**

---

## 🔹 Real-World Use Case (Ecto-style Query)

Keyword lists are commonly used for passing **query conditions**:

```elixir
find_where([
  age: {:>, 10},
  subscribed: true
])
```

Meaning:

- age > 10
- subscribed == true

👉 Reads like natural filtering conditions.

---

## 🔹 Special Syntax (Very Important 🚨)

### 1. Omit Square Brackets (if last argument)

```elixir
find_where age: {:>, 10}, subscribed: true
```

👉 Still treated as a **single keyword list argument**

---

### 2. Omit Parentheses (common in Elixir)

```elixir
find_where age: {:>, 10}, subscribed: true
```

👉 Looks like multiple arguments but is actually **one keyword list**

---

## 🔹 Why This Matters

This syntax allows Elixir code to read almost like English:

```elixir
User.find_where age: {:>, 10}, subscribed: true
```

👉 Very clean and expressive

---

## 🔹 Summary

- Keyword lists = **list of tuples**
- Syntax shortcut: `[key: value]`
- Allow **duplicate keys**
- Maintain **order**
- Common for **options & configs**
- Special syntax when used as **last function argument**

---

## 🧠 Mental Model

Think of keyword lists as:

> "Ordered configuration options that are easy to read and flexible"
